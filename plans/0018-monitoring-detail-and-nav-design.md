# 0018 — 모니터링 화면 정보량 확대 + 메인 메뉴 네비게이션 디자인 개선

**날짜**: 2026-07-15

**대상 작업**: 사용자가 선택한 두 화면 — (1) 모니터링 화면, (2) 메인 메뉴/전체 네비게이션 — 에 더 자세한 정보와 시각적 개선을 추가.

**설계**:
- **메인 메뉴**: 박스 테두리(┌─┐│└─┘)로 감싸 시각적으로 구분. 요약 아래에 "할 일 알림"을 추가 — RESERVED(승인 대기) 건수가 0보다 크면 노란색으로, CONFIRMED(출고 대기) 건수가 0보다 크면 청록색으로 "N건 있습니다 (메뉴 X)" 안내를 표시해 다음에 뭘 해야 할지 바로 보이게 함.
- **모니터링 화면**:
  - "주문량 확인" 표에 "비중" 컬럼 추가 — 4개 상태 중 최대 수량 대비 각 상태 수량의 상대적 비율을 `render_bar`로 시각화.
  - "재고량 확인" 표에 "충족률" 컬럼 추가 — `stock_quantity / pending_qty * 100`(부족분 0이면 100%로 처리)을 `render_bar`로 시각화, 재고가 대기 수요를 얼마나 커버하는지 한눈에 파악 가능.

**실행 계획**:
1. `console_view.py`의 `_print_main_menu`/`_print_summary`에 박스 테두리 + 알림 추가 (colors.py에 YELLOW 이미 존재, table.py의 `pad` 재사용).
2. `_monitoring_menu`에 비중/충족률 바 컬럼 추가.
3. 테스트 보강(필요 시), 전체 pytest 통과 확인.
4. 실제 실행으로 시각 확인 후 커밋.

**상태**: ACTION 완료 (박스 메뉴, 승인/출고 대기 알림, 주문량/재고량 표에 비중·충족률 바 추가, 전체 96개 테스트 통과, 실제 실행으로 시각 확인). COMMIT 완료 — `0d9f5ec`.
